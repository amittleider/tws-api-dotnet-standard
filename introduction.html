Introduction

The TWS API is a powerful interface through which IB clients can automate their trading strategies, request market data or handle their portfolio.

Audience

Our API components are aimed at experienced professional developers willing to enhance the current TWS functionality. Regreatably, Interactive Brokers cannot offer any programming consulting. Before contacting our API support, please always refer to our available documentation and samples.

How to use this guide

This guide references the Java, VB, C# and C++ Testbed sample projects to demonstrate the TWS API functionality. All code snippets are extracted from these projects and we suggest all those users new to the TWS API to get familiar with them in order to quickly understand the fundamentals of our programming interface. The Testbed sample projects can be found within the samples folder of the TWS API's installation directory.

Limitations

Our programming interface automates some of the operations a user normally does when working with the TWS such as placing orders, viewing an instrument's live data, etc. There is no logic within the API other than to ensure the integrity of the exchanged messages. Most validations and checks occur in the TWS and our servers. Because of this it is highly convenient to get familiar with the TWS itself in order to gain a better understanding on how our platform works. Before spending precious development time troubleshooting on the API side, it is recommended to first experiment with the TWS directly.

Remember: if a certain feature or operation is not available in the TWS, it will not be available on the API side either!

Requests

The TWS is designed to accept up to fifty messages per second coming from the client side. Anything coming from the client application to the TWS counts as a message (i.e. requesting data, placing an order, requesting your portfolio, etc.). This limitation is applied to all connected clients in the sense were all connected client applications put together cannot exceed this number. On the other hand, there are no limits on the amount of messages the TWS can send the client application.
