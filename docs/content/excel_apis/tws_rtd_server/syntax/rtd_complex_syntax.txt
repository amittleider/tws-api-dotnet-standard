/*! \page rtd_complex_syntax Complex Syntax		Complex Syntax provides the most flexibility that it allows users to customize all formula strings individually, where each string only represent one single parameter. There is no rule for the sequence of the appearance of each parameter.		<DFN class="rtd_formula">	=RTD(<b>ProgID</b>, <b>Server</b>, <b>String1</b>, <b>String2</b>, <b>String3</b>...)	</DFN>		where		 - <b>ProgID</b> = "Tws.TwsRtdServerCtrl"	 - <b>Server</b> = \"\" (empty string)		For exmaple, the formula below will request the Ask Size for IBM@ISLAND:		<DFN class="rtd_formula">	=RTD("Tws.TwsRtdServerCtrl",,"sym=IBM", "sec=STK", "exch=ISLAND", "qt=AskSize") 	</DFN>		The formula below will request the Bid price for IBM:		<DFN class="rtd_formula">	=RTD("Tws.TwsRtdServerCtrl",,"sym=ES", "sec=FUT", "exch=GLOBEX", "cur=USD", "exp=201712", "qt=Bid") 	</DFN>		\section rtd_complex_syntax_strings Complex Syntax Strings		See table below for a full list available Complex Syntax strings:		<table>	<tr><th>Name</th><th>String Syntax</th><th>Description</th></tr>	<tr><td>Contract ID</td><td>"conid="</td><td>The unique contract ID generated by IB. Can be found at TWS \ref contracts "Contract Description" .</td></tr>	<tr><td>Symbol</td><td>"sym="</td><td>The contract symbol.</td></tr>	<tr><td>SecurityType</td><td>"sec="</td><td>The type of security, e.g. 'STK', 'FUT' and so on.</td></tr>	<tr><td>LastTradeDateOrContractMonth</td><td>"exp="</td><td>Format 'YYYYMMDD' is used for defining the Last Trade Date, while format 'YYYYMM' is used for defining the Contract Month.</td></tr>	<tr><td>Strike</td><td>"strike="</td><td>The strike price for an option contract.</td></tr>	<tr><td>Right</td><td>"right="</td><td>'C' or 'P' for an option contract.</td></tr>	<tr><td>Multiplier</td><td>"mult="</td><td>The contract multiplier.</td></tr>	<tr><td>Exchange</td><td>"exch="</td><td>The exchange where to get market data from. For equities, 'SMART' means top data from all possible exchanges.</td></tr>	<tr><td>PrimaryExchange</td><td>"prim="</td><td>The primary exchange of the contract. It is mostly specified when an contract ambiguity occurs for equity symbols that are listed on multiple exchanges.</td></tr>	<tr><td>Currency</td><td>"cur="</td><td>The currency the contract is traded in.</td></tr>	<tr><td>LocalSymbol</td><td>"loc="</td><td>The local symbol of the contract. Note the Local Symbol is mostly used for futures and options, and is different from the Symbol.</td></tr>	<tr><td>TradingClass</td><td>"tc="</td><td>The trading class of the contract.</td></tr>	<tr><td>Combo</td><td>"cmb="</td><td>Combo contract has to be defined using Complex Syntax or Mixed Syntax. The syntax for defining the combo is: <br><br>"cmb=<conid1>#<ratio1>#<action1>#<exchange1>;<conid2>#<ratio2>#<action2>#<exchange2>;"<br><br> ,where combo legs are separated by ';' and individual leg parameters are separated by '#'. See more \ref rtd_samples_spread "Spread Samples" .</td></tr>	<tr><td>UnderComp</td><td>"und="</td><td>Delta-Neutral Underlying Component. The syntax for defining the undercomp is: <br><br>"und=<<conid>#<delta>#<price>"<br><br>, where undercomp parameters are separated by '#'.</td></tr>	<tr class="gray"><td>MktDataOptions</td><td>"opt="</td><td>Currently not supported.</td></tr>	<tr><td>GenericTickList</td><td>"genticks="</td><td>A comma separated Ids of available Generic Tick Types.</td></tr>	<tr><td>Topic</td><td>"qt="</td><td>Topic of market data request.</td></tr>	<tr><td colspan="3" class="separator"></td></tr>	<tr><td>Host</td><td>"host="</td><td>Host IP address.</td></tr>	<tr><td>Port</td><td>"port="</td><td>Socket port.</td></tr>	<tr><td>ClientId</td><td>"clientid="</td><td>The client ID for socket connection. Note that the client ID is used for identify multiple simultaneous API connections to the same TWS. It was originally designed for API users who would like to manage their strategies separately from different API programs. Since the TWS RTD Server API is currently only supported for real-time market data, there is no need to use multiple client IDs.</td></tr>	</table>		\section rtd_complex_additional Additional Info		See also:	 - \ref rtd_simple_syntax	 - \ref rtd_mixed_syntax		<br><br>*/
